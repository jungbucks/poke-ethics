# 포케 딜레마 — 유지보수 플레이북 (후임 개발 모델용 진입점)

> AI 윤리 딜레마 체험 플랫폼. **Moral Machine의 무해화 버전** — 죽음 대신 치료 순서·선발·추천·프라이버시 딜레마.
> 발전 방향과 백로그는 `ROADMAP.md`. 이 문서에 없는 규칙을 임의로 발명하지 말고, 애매하면 사용자에게 질문할 것.

## 구조 (2파일 + 문서)

| 파일 | 역할 | 수정 빈도 |
|---|---|---|
| `dilemmas.js` | **딜레마·축 데이터 전용** — `window.POKE_DILEMMAS` / `window.POKE_AXES`. 스키마는 파일 상단 주석이 원본 | 높음 (콘텐츠 추가는 여기만) |
| `index.html` | 엔진 (화면 3개 상태 머신 + 로드·검증·축 균형 추첨·토큰 치환 + PokeAPI 캐스팅 + 리포트) | 낮음 (기능 추가 시만) |
| `CLAUDE.md` / `ROADMAP.md` | 이 문서 / 발전 방향 | 구조 바꾸면 반드시 갱신 |

- 순수 HTML/CSS/JS, 빌드 도구·npm 금지. 더블클릭(file://)으로도 동작해야 함.
- 배포: GitHub Pages (`jungbucks/poke-ethics`) → 정글(jgle.kr) 앱스토어에 웹앱 카드로 노출.

## 절대 규칙 (이 프로젝트의 정체성 — 위반 금지)

1. **무해 원칙**: 죽음·살해·사망을 다루는 딜레마 금지. `loadDilemmas()`가 금칙어(`죽음|죽는|죽이|사망|살해|자살`)를 자동 검열해 제외한다 — **이 검열 코드를 약화시키지 말 것.** 이 플랫폼은 Moral Machine이 교실에 부적절(죽음 소재)해서 만든 것이다.
2. **정답 없음 원칙**: 어떤 선택지도 명백한 오답으로 쓰지 말 것. 양쪽 모두 변호 가능해야 하고, 리포트·문구 어디에도 특정 성향을 우월하게 표현하지 말 것.
3. **딜레마 추가 = `dilemmas.js`만 수정.** 엔진(index.html)을 건드리는 순간 그 이유를 커밋 메시지에 남길 것. 스키마 검증(필수 필드·pole·축·id 중복)은 로드 시 자동 — 콘솔 경고 0을 유지.
4. **개인정보**: 이름·학번은 복사 텍스트에만 쓰고 **localStorage 등에 저장하지 않는다**.
5. **PokeAPI 의존은 항상 폴백과 쌍으로**: 모든 네트워크 기능은 실패 시 내장 데이터(FALLBACK 12마리·EMBED_DILEMMAS)로 완주 가능해야 한다. 타임아웃 4.5초 유지.
6. 포켓몬은 비상업 교육 목적으로만. 스프라이트는 PokeAPI 공식 아트워크 URL만 사용, 이미지 파일을 저장소에 넣지 말 것.

## 아키텍처 (확장 지점)

```
dilemmas.js (데이터)          index.html (엔진)
POKE_AXES ──────────┐        loadDilemmas()  ← 검증·무해 검열
POKE_DILEMMAS ──────┼──────▶ pickDilemmas()  ← 축 균형 추첨 (풀이 커져도 한 판 7개)
  {mN} 토큰 ────────┘        buildRounds()   ← 캐스트 치환
                             buildCast()     ← PokeAPI 랜덤 소환 (실패 시 FALLBACK)
                             화면: scr-title → scr-round × N → scr-report
```

- **딜레마 풀을 늘리면** 추첨이 자동으로 리플레이 가치를 만든다 (같은 판에 같은 딜레마 중복 없음, 축별 고르게).
- **새 축 추가** = POKE_AXES에 키 추가 + 그 축의 딜레마 2개 이상 (1개면 리포트가 빈약해짐).
- **화면 추가** = `.screen` 섹션 + `show(id)` 호출. 상태 머신은 이 함수 하나로 관리된다.

## 작업 절차 (모든 변경 시)

1. 수정 → 브라우저에서 열어 콘솔 경고 0 확인.
2. **헤드리스 프로브 검증** (가장 저렴하고 확실): iframe으로 index.html을 감싸고 시작→선택→7라운드→리포트→복사를 자동 클릭하는 probe.html을 임시 생성해 `msedge --headless=new --allow-file-access-from-files --virtual-time-budget=30000 --dump-dom`으로 결과 <pre> 확인 후 probe 삭제. (git log의 초판 커밋에 프로브 전문이 있음 — 550e902 참고)
3. 커밋 메시지는 한국어로, 무엇을·왜. 푸시는 사용자에게 확인.
4. 콘텐츠(딜레마) 추가 시 셀프 체크: ① 무해한가 ② 양쪽 다 변호 가능한가 ③ 실제 AI 윤리 사례와 연결되는가(rf 필드) ④ 중고생이 1분 안에 이해하는가.
