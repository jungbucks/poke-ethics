## 1. 연구 배경과 차별점

- 이론적 뿌리: 트롤리 딜레마 → **Moral Machine** (Awad et al., 2018, *Nature*)이 자율주행이라는 옷을 입혀 대중화.
  본 연구는 그 옷을 한 번 더 갈아입힌 것 — **포켓몬**이라는 옷. 검증된 강제 이지선다 프레임워크의 차용으로 설계 정당성을 확보한다.
- 선행 지형: 게이미피케이션→참여·몰입 효과는 다수 검증됨(닦인 도로). 게이미피케이션+도덕 딜레마도 보드게임형 선례 존재.
  **포켓몬 API를 학습 설계 요소로 쓰는 연구는 거의 없음 → 본 연구의 기여점.**
- **차별점 (서론에 반드시 명시)**: Moral Machine의 복제가 아니다. 진짜 질문은
  ① 죽음을 제거한 무해 딜레마에서도 판단 패턴이 나타나는가
  ② **윤리적으로 무관한 지각 속성(귀여움·강함·희귀함)이 분배 판단에 개입하는가**
  ③ 포켓몬 API 게이미피케이션이 일반 게이미피케이션·비게이미피케이션 대비 차이를 만드는가.
- **"왜 포켓몬 API인가"에 대한 답 (논문적 정당성의 핵심)**: 단순 흥미 유발 장식이 아니라,
  포켓몬의 속성 체계(타입·희귀도·진화 단계·크기)를 **딜레마의 실험 변수로 직접 구조화**하는 데 쓴다 (§3).
- 무해 원칙(죽음 소재 금지)은 연구에서도 유지 — 미성년 대상 연구윤리(기관 심의) 관점에서 강점.

## 2. 측정 구인의 명확화 (가장 취약한 지점 — 정면 대응)

**이 플랫폼이 측정하는 것은 "윤리 의식" 전체가 아니라 "윤리적 판단 성향(가치 지향)"이다.**

| 구인 | 측정 도구 | 역할 |
|---|---|---|
| 윤리적 **판단 성향** (효율↔형평 등 가치 지향, 속성 편향) | 본 플랫폼의 시행 로그 (선택·RT) | 주 분석 — 판단 "패턴" |
| 윤리 **의식/판단력** (민감성·판단력·행동의지) | **기존 검증 척도** — DIT-2/KDIT(한국판 도덕판단력검사) 등, 사전-사후 지필 | 효과 검증 + 플랫폼 점수와의 상관 = **준거 타당도** 근거 |

- 논문에서 변수 명명 주의: 플랫폼 산출물을 "윤리 의식 점수"로 부르지 않는다. 성향과 의식의 간극은 이 이원 측정으로 메꾼다.
- 연구 질문 이층화: (a) 처치 중 **판단 패턴**은 조건별로 어떻게 다른가 (b) 처치 전후 **윤리 의식 검사 점수**는 변하는가.

## 3. 조작적 정의 — PokeAPI 필드 매핑 (방법론적 독창성)

| 변수 | 조작적 정의 (PokeAPI) | 비고 |
|---|---|---|
| 귀여움 | `is_baby`, `height/weight`(소형) 후보 풀 → **파일럿 평정으로 타당화** | 학생 30명 내외, 7점 척도. API 지표-평정 상관 자체가 부록 거리 |
| 강함 | `stats` base stat 총합, `evolution-chain` 진화 단계 | 총합 상·하위 사분위로 상/하 풀 |
| 희귀도 | `is_legendary`, `is_mythical` | 이지적 정의 |
| 수 | 시나리오 내 개체 수 (1 vs 5) | 텍스트 조작 |
| 통제 | 부상 정도 등 윤리적 조건은 시나리오 텍스트로 양쪽 동일 고정 | 속성 쌍만 변함 |

## 4. 혼입변수 통제 (외부 검토 지적 1순위)

현 v1.0의 **매판 랜덤 소환은 연구 모드에서 폐기**한다 — 개체 매력도가 어느 선택지에 걸리느냐가 통제 불가능해,
게이미피케이션 효과와 개체 매력도 효과가 분리되지 않는다. 대책 3중:

1. **고정 캐스트 세트**: research 모드는 런타임 랜덤 소환 대신, **사전 타당화된 고정 포켓몬 세트**(research-config.js)를 사용.
   PokeAPI는 세트 "구축 단계"의 속성 추출 도구로 쓰고, 세션 중에는 고정 자산처럼 동작한다.
2. **사전 평정 공변량**: 파일럿에서 세트 전체의 매력도·친숙도를 평정받아 공변량(또는 짝맞춤 기준)으로 사용.
   본검사 참가자에게도 간단 친숙도 문항(각 포켓몬을 아는가) 포함.
3. **상쇄 균형(counterbalancing)**: 같은 딜레마에 포켓몬 쌍의 역할(A/B)과 좌우 위치를 참가자 간 교차 배치 —
   개체 효과가 조건 간에 상쇄되도록.

## 5. 연구 질문과 가설

| 가설 | 내용 | 대응 축 |
|---|---|---|
| **H1 귀여움 편향** | 윤리 조건 동일 시 지각된 귀여움이 높은 대상 우선 선택 | (신규 — 독창 지점) |
| **H2 강함/유용성 편향** | 전투력·진화 단계 높은 대상 우선 선택 | Moral Machine의 사회적 지위 축과 평행 |
| **H3 희귀도 편향** | 레전더리 > 일반 | (신규) |
| **H4 수(數) 판단** | 다수(5) vs 소수(1) — **Awad et al. 인간 대상 결과와 직접 대조** | 공리주의 축 |
| **H5 처치 조건 효과** | 3집단(§6) 간 판단 패턴·사후 검사 점수 차이 | (신규 — 효과 분리) |

- 타당도 프레임: "포켓몬 속성이 생명의 무게를 대변하는가"를 방어하지 않는다. 질문을 뒤집는다 —
  **"무관한 속성이 판단에 개입하는가."** 개입의 존재 자체가 발견 (moral circle / mind perception 문헌 접속).

## 6. 실험 설계

- **집단 설계 (between, 3집단)** — "포켓몬 API의 효과"를 분리하기 위한 최소 구성:
  ① **포켓몬 게이미피케이션** (본 플랫폼) ② **일반 게이미피케이션** (동일 딜레마 + 포인트·수집 UI, 포켓몬 없이 중립 아이콘)
  ③ **텍스트 전용** (동일 딜레마, 게임 요소 0)
- **딜레마 설계 (within)**: 피험자 내 반복 측정, 라틴 방격으로 순서 통제, 좌우 무선화(§4-3), 프레이밍 라벨·성찰 문구 숨김.
- **문항 수 (심리측정 요건 — 외부 검토 지적 2순위)**: 축당 2문항(현행)으로는 내적 일관성(Cronbach's α) 산출 불가 →
  research 모드는 **축당 4~5문항, 총 12~15라운드**로 확장 (ROADMAP v1.1 딜레마 풀 확장과 정합).
  15라운드 × 약 40초 ≈ 10분 — 중·고생 집중 시간 안. 신뢰도·문항 분석은 파일럿에서 선행.
- **표본**: 학급/학년 단위. 3집단 × 검정력 분석으로 최소 표본 산출 (사용자 확정 사항 — 미정).

## 7. 측정과 데이터 파이프라인

| DV | 수집 | 용도 |
|---|---|---|
| 선택 (A/B) | 시행 로그 | 조건·속성별 선택률 (주 분석) |
| 반응 시간 | 제시~클릭 ms | 판단 갈등 지표 |
| 확신도 (옵션) | 선택 후 4점 | 민감도 보조 |
| 윤리 의식 | 플랫폼 밖 사전-사후 DIT-2/KDIT | 효과 검증 + 준거 타당도 |

- 시행 로그 스키마: `{참가자코드(익명), 집단, 세트/순서 배치, 딜레마id, 속성조건, 좌우배치, 선택, RT}` —
  **개인 식별 정보 저장 금지**(참가자 코드-학번 매칭표는 교사가 오프라인 별도 보관).
- 수집 경로(서버리스): 종료 시 기계 판독 한 줄 인코딩 → **구글폼 사전 채움 URL 원클릭 제출** + 복사 폴백 →
  `teacher.html` 파서가 일괄 CSV 변환 → 통계 패키지.

## 8. research 모드 스펙 (후임 구현 지침 — 설계 확정 후 착수)

- 진입: `?mode=research&group=poke|gami|plain&set=1..k` — **수업 모드(기본)와 한 코드베이스, 세 처치 변형**:
  `poke`(포켓몬) / `gami`(중립 아이콘+포인트·수집 UI) / `plain`(텍스트 전용).
- research 공통: 이유 라벨·rf 숨김 / 좌우 무선화 / RT·시행 로깅 / 리포트 대신 "참여 완료+제출" 화면 / 라운드 12~15.
- **캐스팅: 런타임 랜덤 금지** — `research-config.js`(신설)의 고정 세트·상쇄 배치표를 따른다. PokeAPI 실패 시 연구 세션은
  중단 안내 (폴백 진행 금지 — 조작 무결성). 세트 구축용 속성 추출 스크립트는 별도 유틸로.
- 무해 검열·스키마 검증은 연구 딜레마에도 동일 적용.
